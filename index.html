Install Claude Code
with Figma MCP

Get your local environment wired up in under 5 minutes. This guide walks you through installing Claude Code and connecting the Figma MCP server so you can send live UI directly to Figma.

~5 min setup macOS · Linux · Windows

00

Prerequisites

Before starting, make sure you have the following in place.

👤
Anthropic Account
Claude Pro or Max plan, or API credits
🎨
Figma Account
Any paid plan for MCP access
💻
Terminal
Bash, Zsh, PowerShell, or CMD
🌐
Internet
Required for install and auth
ℹ️

Windows users: Claude Code requires Git for Windows or WSL. Install Git for Windows first, then use the PowerShell command in Step 01.

01

Install Claude Code

Anthropic now ships a native binary installer — the recommended method. No Node.js dependency, auto-updates in the background. Pick your platform below.

bash 
# Install Claude Code (native binary — recommended)
curl -fsSL https://claude.ai/install.sh | bash
powershell 
# Run in PowerShell (not CMD)
irm https://claude.ai/install.ps1 | iex
bash 
brew install claude-code
# Note: Homebrew does NOT auto-update. Run periodically:
brew upgrade claude-code
bash 
# Requires Node.js 18+. Deprecated — use native installer when possible.
# Never use sudo npm install -g
npm install -g @anthropic-ai/claude-code
⚠️

Never run sudo npm install -g — it creates permission issues. If you see EACCES errors, configure a user-level npm directory instead. See the troubleshooting note at the bottom of this guide.

Once installed, verify it's working:

terminal 
$ claude --version
Claude Code 2.x.x
# A version number means you're good to go
💡

If you see command not found, open a new terminal window. The installer adds Claude to your PATH, but the current session needs to reload. If it persists, run source ~/.zshrc or source ~/.bashrc.

02

Add the Figma MCP Server

The Figma MCP server connects Claude Code to your Figma workspace. You can add it scoped to a single project or globally across all projects.

📁
Project-scoped
Only active in the current directory
bash 
claude mcp add --transport http figma https://mcp.figma.com/mcp
🌍
Global (recommended for teams)
Available across all your projects
bash 
claude mcp add --scope user --transport http figma https://mcp.figma.com/mcp
ℹ️

If the Figma server doesn't appear after adding it, restart Claude Code completely. MCP connections initialize at startup, so adding a new server mid-session requires a restart.

03

Authenticate with Figma

Once Claude Code is running, complete the OAuth flow to connect your Figma account.

1
Open Claude Code and run /mcp

Type /mcp in any Claude Code session. A list of connected servers appears.

2
Select the Figma server

From the list, highlight figma (http, remote) and press Enter.

3
Choose Authenticate

Select Authenticate from the menu. Your browser will open the Figma OAuth page.

4
Complete the OAuth flow

Log in to Figma in your browser and grant access. Return to Claude Code — you should see a confirmation.

claude code 
❯ /mcp
  figma (http, remote)  [Press Enter]
  → Authenticate
  ✓ Authentication successful. Connected to figma
✅

Run /mcp again — the figma server should show a green status. That means you're fully connected.

04

Send Your UI to Figma

With the MCP connected, you can capture any live, running UI from your browser — localhost, staging, or production — and send it as editable Figma layers.

1
Build something and preview it in the browser

Run your dev server as usual. Navigate to the screen you want to capture.

2
Type "Send this to Figma" in Claude Code

Claude uses the generate_figma_design tool to capture the rendered UI from your browser. A capture toolbar appears — select the full screen or specific elements.

3
Find your frames in Figma

The captured UI appears as editable frames in your Figma workspace. Text, buttons, and layout are all live Figma components with auto-layout.

⚠️

Note: The generate_figma_design tool is currently Claude Code-only and Remote MCP-only. The Desktop MCP server does not support this direction.

💡

You can capture multiple screens in a single session. Figma preserves the sequence across frames, so multi-screen flows stay coherent as a set of related layouts.

05

# Captures your live browser UI as editable Figma layers
🛠 Troubleshooting
!
Figma server missing after adding it

Fully restart Claude Code. MCP connections initialise at startup.

!
npm EACCES permission error

Don't use sudo. Run mkdir ~/.npm-global && npm config set prefix '~/.npm-global' and add it to your PATH, then reinstall.

!
claude: command not found

Open a new terminal, or run source ~/.zshrc (macOS/Linux). On Windows, restart PowerShell.
